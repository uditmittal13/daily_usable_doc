## Steps
1st step:-
   Deploy the crd yaml file ( custom resource definiton for mongodbcomunity.mongodb)
2nd step:-
    created separate new namespace for deploying mongok8operator and mongo cluster
3rd Step:-
## Define rbac yamls for your cluster and deploy it in specific namesapce. ( where directory contains kcustomize.file in which each yaml there).
   kubectl apply -k directory_name
   
4th Step:-
## Readiness of mongomanager.yaml ( i.e deployment file for new operator setup, having all details of image pulling/ opertor name/ which namespace & pod to watch by operator/
agent image/ readiness probe /mongo db image /mongo repo url/image pull policy/resources details like cpu and memory/ service account name)
   Be careful while you deploy this file, dont forget to mention it deploy in specifc namespace which you are targeting for your cluster
   kubectl apply -f mongomanager.yaml -n namespace
   
5th step:-  (
## Readiness of yaml to setup the mongo cluster ( which describes no of replicas, namespace, version, secuirty authentication-scram,users[passwordsecretRef,scram credentials])
   mongo-cluster.yaml
   
6th step:-
## create secret for mongo-admin-password.  ( Must step)
kubectl create secret generic mongo-admin-password --from-literal=password=Admin@123 -n mongod2

Note:- All Required yamls are attached here in the same folder.
